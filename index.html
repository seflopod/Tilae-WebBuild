Contributions and Considerations

Design Considerations

  • "Easy to learn, difficult to master" board game
  • Increased accessibiity for players with vision impairments
  • An AI that is not a master of the game

Notable Systems

Audio

In order to better accomodate visually impaired players, we needed to make use of audio cues to communicate UI and game states. We listed what that would mean:

  • Interactable portions of the UI need audio cues when a pointer enters.
  • Any state change on the UI needs to communicate the nature of that change.
  • Since the game board is made of multiple cells, each cell needs to communicate its state via unique sounds.
  • Each player needs a unique sound that can be used as a cue for turn order.

For the UI we decided voiceover would be the best, while for the playable portions we decided to use a base note for each cell on the board which would be modified by the cell's state. As a result of these decisions we found we needed more sounds than usual for a project of this size which meant that we needed a means of easily managing audio.

I wrote a simple audio managment system to accomplish this goal. The system is built on a custom asset (SoundLibrary) which contains a Dictionary with custom serialization. The SoundLibrary can be queried for a sound by name so by adhering to a solid naming convention finding sounds in code is easy. A sound is then applied to a relevant AudioSource and played once found.

AI

We decided on creating a simple AI for play, as the general intent was for this to be a multiplayer game. On a high level, the AI works by doing one of the following:

  1. If the game is in the place players state, place the piece for the agent.
  2. If the game is in the initial plase tiles phase, place a tile in a random cell.
  3. If there is only one other player alive that can be eliminated by placing a tile, use the place tile aciton.
  4. If there is only one open cell around the agent, move to that cell.
  5. Any other situation is the normal course of play. Determine a score between 0 and 1 (inclusive) for each of the following actions and choose the one with the highest score:
    • Place a tile. The best possible cell for tile placement is determined based on the following assumptions:
      • It is more desirable to place a tile next to an opponent with as few open cells around it as possible.
      • It is more desirable to place a tile next as many opponents as possible.
      • It is less desirable to place a tile next to the agent.
    • Move. The best possible move is based on the following:
      • The increase in blank cells surrounding the agent by moving from the current position to a new position.
      • The desirablity of having certain cells around the agent blank (cell weight).
    • Push tiles. Rather than spend time determining the best way to score tile pushing, we decided to make this a random choice.

The AI has some shortcuts for repeating moves too frequently which prevents stalemates when two or more agents are playing against each other. The AI does not analyze much beyond the cells immediately surrounding it in order to ease development speed.
